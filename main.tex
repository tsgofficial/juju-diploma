\vspace{0.6cm}

\noindent\textbf{\textit{Түлхүүр үгс: IoT, ESP32, ESP32-CAM, ухаалаг тэжээгч, stepper мотор, насос, Firebase Realtime Database, Cloudinary, web interface, алсын хяналт}}
\vspace{0.5cm}

\noindent Орчин үеийн амьдралын хэв маяг, ажил болон хичээлийн ачааллаас шалтгаалан гэрийн тэжээвэр амьтны хоол болон усны арчилгааг тогтмол хийхэд хүндрэл үүсдэг. Энэ асуудлыг шийдвэрлэх нэг боломжит арга нь интернетэд холбогдох чадвартай, автоматаар ажиллах, алсаас хянах боломжтой ухаалаг төхөөрөмж боловсруулах явдал юм. Энэхүү дипломын ажлын хүрээнд IoT технологид суурилсан гэрийн тэжээвэр амьтны ухаалаг хооллолтын системийг боловсруулж, хэрэгжүүлсэн.

Систем нь ESP32 микроконтроллер, ESP32-CAM, stepper мотор, усны насос, RTC модуль, web интерфейс, Firebase Realtime Database болон Cloudinary cloud storage гэсэн үндсэн хэсгүүдээс бүрдэнэ. ESP32 нь хоол болон ус өгөх команд, автомат цагийн логик, hardware удирдлагыг гүйцэтгэх бол ESP32-CAM нь камерын зураг авч Cloudinary-д хадгална. Харин Firebase Realtime Database нь command, status, frame URL, timestamp зэрэг өгөгдлийг realtime байдлаар хадгалж, web систем болон төхөөрөмжийн хоорондын мэдээлэл солилцоог хангана.

Төслийн явцад IoT системийн онол, ижил төрлийн smart pet feeder төхөөрөмжүүд, микроконтроллерийн сонголт, моторын төрлүүд, cloud өгөгдөл хадгалалтын шийдлүүдийг харьцуулан судалсан. Судалгааны үндсэн дээр Wi-Fi холболттой, өргөтгөх боломжтой, өртөг харьцангуй бага, web болон cloud системтэй холбогдох боломжтой ESP32 суурьтай шийдлийг сонгосон.

Хэрэгжилт ба туршилтын шатанд feed command, drink command, camera frame upload, Firebase status update, web interface-ийн ажиллагааг туршсан. Туршилтын үр дүнгээс харахад систем нь хэрэглэгчийн web interface-ээс өгсөн командын дагуу хоол болон ус өгөх механизмыг ажиллуулж, камерын зургийг cloud орчинд хадгалан web дээр харуулах боломжтой болсон.

\chapter{Судалгааны хэсэг}

\section{Судалгааны үндэслэл ба асуудлын тодорхойлолт}

Гэрийн тэжээвэр амьтныг өдөр бүр тогтмол цагт хооллох, цэвэр усаар хангах, эзэн гэртээ байхгүй үед нөхцөл байдлыг нь шалгах нь амьтантай айл өрхийн өдөр тутмын чухал хэрэгцээ юм. Гэвч бодит амьдрал дээр ажил, хичээл, замын түгжрэл, аялал, гэнэтийн ажил зэрэг шалтгаанаас болж амьтны хоол болон усыг яг тогтсон цагт нь өгөх боломж үргэлж байдаггүй. Ийм үед зөвхөн нэг удаагийн автомат сав биш, харин интернетээр холбогдож, хэрэглэгч алсаас хянаж, шаардлагатай үед команд өгөх боломжтой систем илүү тохиромжтой гэж үзсэн.

Энэ төслийг сонгосон шалтгаан нь IoT технологийг өдөр тутмын бодит асуудалтай холбож, hardware, software, cloud service, web interface гэсэн олон хэсгийг нэг цогц систем болгон хэрэгжүүлэх боломжтой байсанд оршино. Өөрөөр хэлбэл, энэ ажил нь зөвхөн мотор ажиллуулах туршилт биш, харин хэрэглэгч төхөөрөмжтэй интернетээр холбогдож, өгөгдөл солилцож, камерын мэдээлэл харах боломжтой жижиг хэмжээний IoT систем юм.

Төслийн үндсэн асуудлыг дараах байдлаар тодорхойлсон. Нэгдүгээрт, тэжээвэр амьтанд хоол өгөх ажиллагааг тогтмол, давтагдах боломжтой болгох шаардлагатай. Хоёрдугаарт, ус өгөх үйлдлийг хоолноос тусдаа удирдах хэрэгтэй. Гуравдугаарт, хэрэглэгч төхөөрөмжийн ойролцоох байдлыг алсаас харах боломжтой байх нь итгэлцлийг нэмэгдүүлнэ. Дөрөвдүгээрт, web интерфейс болон cloud өгөгдлийн сан ашиглан төхөөрөмжийн command, status, camera frame мэдээллийг хадгалах шаардлагатай.

\section{Судалгааны зорилго, зорилтууд}

Энэхүү дипломын ажлын ерөнхий зорилго нь IoT технологид суурилсан, гэрийн тэжээвэр амьтанд хоол болон усыг автоматаар өгөх, web интерфейсээр алсаас удирдах, камерын зураг ашиглан нөхцөл байдлыг харах боломжтой ухаалаг хооллолтын систем боловсруулахад оршино.

Энэхүү зорилгын хүрээнд дараах зорилтуудыг дэвшүүлсэн.

\begin{enumerate}[leftmargin=1.3cm]
  \item IoT системийн онолын үндэс, ухаалаг тэжээвэр амьтны тэжээгч төхөөрөмжүүдийн хөгжлийн чиг хандлагыг судлах.
  \item Ижил төрлийн төхөөрөмжүүдийн функц, давуу тал, сул талыг харьцуулж, өөрийн системд хэрэгжүүлэх боломжтой шийдлүүдийг тодорхойлох.
  \item ESP32, ESP32-CAM, stepper мотор, насос, RTC, Firebase, Cloudinary зэрэг бүрэлдэхүүнүүдийг сонгох үндэслэлийг гаргах.
  \item Хоол болон ус өгөх механик ажиллагааг тусдаа удирдах hardware зохион байгуулалт боловсруулах.
  \item ESP32-CAM-аар авсан зургийг Cloudinary-д хадгалж, URL-ийг Firebase Realtime Database-д бүртгэх cloud архитектур хэрэгжүүлэх.
  \item Web интерфейсээр feed, drink, camera, status, health мэдээллийн хэсгийг нэгтгэн зохион байгуулах.
  \item Системийн туршилтыг хийж, үр дүнг хүснэгт, тайлбар, дүгнэлтээр боловсруулах.
\end{enumerate}

\section{IoT системийн онолын үндэс}

IoT буюу Internet of Things нь физик төхөөрөмжүүдийг интернет сүлжээнд холбож, мэдрэгч болон удирдлагын өгөгдлийг цуглуулах, дамжуулах, боловсруулах, хэрэглэгчид хүргэх технологийн ойлголт юм. IoT систем нь ерөнхийдөө мэдрэх хэсэг, боловсруулах хэсэг, сүлжээний холболт, өгөгдөл хадгалах хэсэг, хэрэглэгчийн интерфейс гэсэн таван үндсэн давхаргаас бүрддэг.

Энэхүү төслийн хувьд мэдрэх болон хяналтын хэсэгт ESP32-CAM, RTC болон төхөөрөмжийн төлөвийн мэдээлэл орно. Боловсруулах хэсэг нь ESP32 микроконтроллер бөгөөд хэрэглэгчийн command, цагийн мэдээлэл, мотор болон насосын ажиллагааг удирдана. Сүлжээний холболт нь Wi-Fi-аар хийгдэнэ. Өгөгдөл хадгалалтын хэсэг нь Firebase Realtime Database болон Cloudinary cloud storage-оор шийдэгдсэн. Харин хэрэглэгчийн интерфейс нь web системээр хэрэгжсэн.

\myimage{iot_structure.png}{IoT системийн ерөнхий бүтэц}

IoT системийн гол онцлог нь төхөөрөмжүүд хоорондоо өгөгдөл солилцож, хэрэглэгчийн оролцоог багасгахын зэрэгцээ хяналт, удирдлагыг илүү уян хатан болгодогт оршино. Энэ төслийн хувьд хэрэглэгч гэртээ байхгүй байсан ч web интерфейсээр төхөөрөмжийн төлөвийг харах, feed эсвэл drink command өгөх, камерын хамгийн сүүлийн зургийг шалгах боломжтой. Иймээс энэхүү систем нь IoT-ийн ``алсын хяналт'', ``өгөгдөл солилцоо'', ``автоматжуулалт'' гэсэн үндсэн шинжүүдийг агуулж байна.

\section{Ухаалаг тэжээвэр амьтны тэжээгч төхөөрөмжүүдийн судалгаа}

Судалгааны явцад commercial буюу зах зээл дээр худалдаалагдаж буй ухаалаг pet feeder төхөөрөмжүүд болон сургалт, туршилтын зориулалттай open-source төслүүдийг харьцуулан авч үзсэн. Зах зээл дээрх төхөөрөмжүүдийн нийтлэг функц нь цагийн хуваарийн дагуу хоол өгөх, гар утасны аппликейшнээр удирдах, зарим тохиолдолд камер, микрофон, feeding history, notification зэрэг нэмэлт боломжуудтай байх явдал юм.

\section{ESP32 болон ESP32-CAM модулийн судалгаа}

ESP32 нь Wi-Fi болон Bluetooth холболттой, 32-bit микроконтроллер бөгөөд IoT системийн prototype болон бүтээгдэхүүний эх загварт өргөн ашиглагддаг. Олон GPIO pin, PWM, ADC, I2C, SPI, UART зэрэг интерфейстэй учраас олон төрлийн мэдрэгч, actuator, module-ийг нэг төхөөрөмжид холбох боломжтой. Энэ төслийн хувьд ESP32 нь feed command, drink command, автомат цагийн шалгалт, stepper motor болон насосын удирдлагыг гүйцэтгэж байна.

ESP32-CAM нь ESP32 суурьтай камерын модуль бөгөөд OV2640 камер, Wi-Fi холболт, жижиг хэмжээтэй бүтэцтэй. Энэ модуль нь тэжээвэр амьтны хооллох орчныг харах, авсан frame зургийг web системд дамжуулахад тохиромжтой. Гэхдээ ESP32-CAM нь санах ой болон power тал дээр мэдрэмтгий тул зураг хадгалалтыг төхөөрөмж дээр биш Cloudinary cloud storage дээр хийх шийдлийг сонгосон. Энэ нь төхөөрөмжийн local storage ачааллыг багасгаж, web системд зураг харуулахад илүү тохиромжтой болсон.

\section{Усны системийн шийдлийн судалгаа}

Ус өгөх хэсэгт жижиг DC насос ашиглах шийдлийг сонгосон. Усны системийн хувьд servo эсвэл stepper motor ашиглах шаардлагагүй, учир нь усны урсгалыг бий болгох үндсэн арга нь насосоор шахах явдал юм. Насосыг тодорхой хугацаанд ажиллуулах замаар гарах усны хэмжээг ойролцоогоор удирдаж болно. Энэ шийдэл нь бүтэц энгийн, угсрахад ойлгомжтой, туршилт хийхэд тохиромжтой.

Усны системийг хоолны системээс тусдаа удирдахаар төлөвлөсөн нь хэрэглэгчийн хувьд илүү уян хатан. Жишээлбэл, амьтан хоол идээгүй байсан ч зөвхөн ус өгөх, эсвэл хоолны дараа усны урсгал ажиллуулах боломжтой. Мөн цаашид усны түвшин мэдрэгч нэмэхэд энэ системийг өргөтгөх боломжтой.

\section{Cloud үйлчилгээний сонголт: Firebase ба Cloudinary}

Энэ төслийн нэг чухал хэсэг нь төхөөрөмжийн мэдээллийг web interface болон cloud орчинтой холбох явдал байсан. Command болон status зэрэг жижиг хэмжээний өгөгдөлд realtime database хэрэгтэй, харин камерын зураг хадгалахад media storage хэрэгтэй. Иймээс Firebase Realtime Database болон Cloudinary-г хамтад нь ашигласан.

Firebase Realtime Database-ийг сонгосон шалтгаан нь өгөгдөл realtime байдлаар шинэчлэгдэх боломжтой, web системээс уншихад хялбар, JSON маягийн бүтэцтэй, ESP32 болон JavaScript web системтэй уялдуулахад тохиромжтой байсан. Энэ системд Firebase нь command, cam\_ip, frame\_url, last\_frame\_ts зэрэг өгөгдлийг хадгална.

Cloudinary-г сонгосон шалтгаан нь зураг хадгалах, upload хийх, хадгалсан зурагт URL үүсгэх, web дээр хялбар харуулах боломжтой media cloud service учраас юм. ESP32-CAM дээр зураг хадгалах local storage хязгаарлагдмал тул авсан frame-ийг cloud руу upload хийж, дараа нь URL-ийг Firebase-д хадгалах нь илүү оновчтой шийдэл болсон.

\section{Судалгааны бүлгийн дүгнэлт}

Энэхүү бүлэгт IoT системийн онолын үндэс, тэжээвэр амьтны ухаалаг тэжээгч төхөөрөмжүүдийн судалгаа, микроконтроллерийн сонголт, моторын төрөл, усны систем, Firebase болон Cloudinary cloud үйлчилгээний хэрэглээг харьцуулан авч үзлээ. Судалгааны үр дүнгээс харахад энэ төслийн хувьд ESP32 нь төв удирдлагад, ESP32-CAM нь камерын хяналтад, stepper motor нь хоол гаргах механизмд, насос нь ус түгээх хэсэгт, Firebase нь realtime өгөгдөл хадгалалтад, Cloudinary нь зураг хадгалалтад хамгийн тохиромжтой гэж үзсэн.

\chapter{Ерөнхий дүгнэлт ба цаашдын хөгжүүлэлт}

\section{Ерөнхий дүгнэлт}

Энэхүү дипломын ажлын хүрээнд IoT технологид суурилсан гэрийн тэжээвэр амьтны ухаалаг хооллолтын системийг судалж, зохион байгуулж, хэрэгжүүлж туршлаа. Судалгааны хэсэгт IoT системийн үндсэн ойлголт, ухаалаг тэжээгч төхөөрөмжүүдийн хэрэглээ, commercial болон open-source шийдлүүдийн давуу сул тал, микроконтроллер болон motor сонголт, Firebase болон Cloudinary cloud үйлчилгээний үүргийг харьцуулсан.

Cloud шийдлийн хувьд Firebase Realtime Database болон Cloudinary-г хамтад нь ашигласан нь энэ төслийн гол онцлог болсон. Firebase нь command, status, frame URL, timestamp зэрэг жижиг хэмжээний realtime өгөгдлийг хадгалсан бол Cloudinary нь ESP32-CAM-аас авсан зураг хадгалах үүрэгтэй ажилласан. Иймээс дипломын ажлын зорилго биелсэн гэж дүгнэж байна.

\section{Төслийн онцлог ба шинэлэг тал}

Энэхүү системийн онцлог нь зөвхөн автомат хоол өгөх төхөөрөмж хийхэд бус, олон технологийг нэгтгэсэн IoT architecture боловсруулахад оршино.

\begin{itemize}[leftmargin=1.2cm]
  \item ESP32 дээр суурилсан hardware control;
  \item ESP32-CAM дээр суурилсан камерын зураг авах хэсэг;
  \item Cloudinary ашигласан camera frame storage;
  \item Firebase Realtime Database ашигласан command/status өгөгдөл солилцоо;
  \item Web interface-ээр feed, drink, camera, health, status хэсгийг нэгтгэсэн хэрэглэгчийн орчин;
  \item Stepper motor болон насосыг тусдаа удирдах механик зохион байгуулалт;
  \item PCB болон 3D корпусын төлөвлөлт.
\end{itemize}

\section{Цаашдын хөгжүүлэлт}

Цаашид энэхүү системийг илүү боловсронгуй болгохын тулд хэд хэдэн чиглэлээр хөгжүүлж болно. Нэгдүгээрт, хоолны хэмжээг илүү нарийвчлалтай хэмжихийн тулд load cell буюу жин мэдрэгч нэмэх боломжтой. Хоёрдугаарт, усны түвшин мэдрэгч нэмснээр ус дууссан эсэхийг илрүүлж, хэрэглэгчид мэдэгдэх боломжтой. Гуравдугаарт, Firebase database-ийн \texttt{history} хэсгийг өргөтгөж өдөр, долоо хоног, сар бүрийн feed болон drink history-г хадгалах боломжтой. Дөрөвдүгээрт, user authentication болон security rules-ийг сайжруулж, зөвхөн эрх бүхий хэрэглэгч удирдах боломжтой болгох шаардлагатай. Тавдугаарт, 3D корпусыг бүрэн хэвлэж, ус гоожихоос хамгаалах, хоол бөглөрөхөөс сэргийлэх, цэвэрлэхэд хялбар болгох зэрэг mechanical improvement хийх боломжтой.

\section{Эцсийн дүгнэлт}

Дүгнэж хэлбэл, энэхүү дипломын ажил нь IoT технологийг гэрийн тэжээвэр амьтны өдөр тутмын арчилгаанд ашиглах нэг бодит шийдлийг боловсруулсан. Судалгааны хэсэгт сонголт бүрийн үндэслэлийг харьцуулж гаргасан, зохион байгуулалтын хэсэгт hardware, cloud, web interface-ийн уялдааг тодорхойлсон, хэрэгжилтийн хэсэгт үндсэн функцуудыг туршиж баталгаажуулсан. Энэхүү ажил нь цаашид мэдрэгчийн өгөгдөл, history dashboard, notification, security, сайжруулсан 3D корпус зэрэг нэмэлт хөгжүүлэлтээр илүү бүрэн бүтээгдэхүүн болох боломжтой.

\subsection*{Г.1. ESP32 үндсэн удирдлагын код}

ESP32 микроконтроллер дээр ажиллаж буй үндсэн программ нь Wi-Fi холболт үүсгэх, Firebase Realtime Database-аас command утга унших, command-ийн дагуу stepper motor болон насосыг ажиллуулах үүрэгтэй.


\begin{lstlisting}[language=C++, caption={ESP32-ийн үндсэн программын кодын жишээ}, label={lst:esp32-main}]
#include <WiFi.h>
#include <Firebase_ESP_Client.h>
#include <Stepper.h>

// Wi-Fi болон Firebase тохиргоо
#define WIFI_SSID "YourWiFiName"
#define WIFI_PASSWORD "YourWiFiPassword"
#define API_KEY "YOUR_FIREBASE_API_KEY"
#define DATABASE_URL "https://your-project.firebaseio.com/"

// Stepper моторын pin тохиргоо
const int stepsPerRevolution = 2048;
Stepper stepperMotor(stepsPerRevolution, 14, 27, 26, 25);

// Насосын pin
const int pumpPin = 12;

FirebaseData fbdo;
FirebaseAuth auth;
FirebaseConfig config;

void setup() {
  Serial.begin(115200);
  pinMode(pumpPin, OUTPUT);
  digitalWrite(pumpPin, LOW);

  // Wi-Fi холболт үүсгэх
  WiFi.begin(WIFI_SSID, WIFI_PASSWORD);
  Serial.print("Wi-Fi-д холбогдож байна");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\nWi-Fi холбогдлоо!");

  // Firebase тохиргоо
  config.api_key = API_KEY;
  config.database_url = DATABASE_URL;
  Firebase.signUp(&config, &auth, "", "");
  Firebase.begin(&config, &auth);
  Firebase.reconnectWiFi(true);

  // Stepper моторын хурд
  stepperMotor.setSpeed(10);
}

void loop() {
  // Firebase-ээс command утга унших
  if (Firebase.RTDB.getString(&fbdo, "/feeder/command")) {
    String command = fbdo.stringData();

    if (command == "feed") {
      feedAction();
      Firebase.RTDB.setString(&fbdo, "/feeder/command", "idle");
      Firebase.RTDB.setString(&fbdo, "/feeder/last_action", "feed");
    }
    else if (command == "drink") {
      drinkAction();
      Firebase.RTDB.setString(&fbdo, "/feeder/command", "idle");
      Firebase.RTDB.setString(&fbdo, "/feeder/last_action", "drink");
    }
  }
  delay(1000);
}

// Хоол өгөх функц
void feedAction() {
  Serial.println("Хоол өгч байна...");
  stepperMotor.step(1024);  // Хагас эргэлт
  delay(500);
}

// Ус өгөх функц
void drinkAction() {
  Serial.println("Ус өгч байна...");
  digitalWrite(pumpPin, HIGH);
  delay(3000);  // 3 секунд насос ажиллуулна
  digitalWrite(pumpPin, LOW);
}
\end{lstlisting}

\section{Firebase Realtime Database-ийн security rules}

Firebase Realtime Database-д хадгалагдсан өгөгдөлд хандах эрхийг security rules-ээр удирдана. Дипломын ажлын туршилтын үе шатанд дараах энгийн rules ашигласан.

\begin{lstlisting}[language=Java, caption={Firebase Realtime Database security rules}, label={lst:firebase-rules}]
{
  "rules": {
    "feeder": {
      ".read": true,
      ".write": true,
      "command": {
        ".validate": "newData.isString() &&
                      (newData.val() == 'feed' ||
                       newData.val() == 'drink' ||
                       newData.val() == 'idle')"
      },
      "frame_url": {
        ".validate": "newData.isString() &&
                      newData.val().beginsWith('https://')"
      },
      "last_frame_ts": {
        ".validate": "newData.isNumber()"
      }
    }
  }
}
\end{lstlisting}

\textbf{Анхааруулга:} Дээрх rules нь зөвхөн туршилтын зориулалттай бөгөөд бодит production орчинд ашиглах боломжгүй. Production орчинд authentication үндсэн дээр зөвхөн эрх бүхий хэрэглэгч read/write хийх боломжтой болгох шаардлагатай.
